# How to Use This Document as a Builder

Read the [Cross-Protocol Synthesis](15-synthesis.md) first for the patterns, then jump to the case studies that match your problem. This table maps common builder situations to the sections worth reading.

| If you are working on... | Read these case studies | Key idea |
|---|---|---|
| Designing a points campaign | [Ethena](protocols/ethena.md), [ether.fi](protocols/etherfi.md), [Spark](protocols/spark.md) | Reward loops and integrations, not raw deposits |
| Graduating off points | [Maple](protocols/maple-finance.md), Aave Merit ([Aave](protocols/aave.md)) | Revenue-backed, targeted incentives replace seasons |
| Cold-starting liquidity | [Lido](protocols/lido.md), [Spark](protocols/spark.md), [Morpho](protocols/morpho.md) | Subsidize the doubt; inherit trust or capital |
| A retroactive airdrop / token launch | [Uniswap](protocols/uniswap.md), [Ethena](protocols/ethena.md), [Spark](protocols/spark.md) | Reward past behavior; keep 50%+ for users |
| Landing a B2B2C embed | [Morpho](protocols/morpho.md), [Aave](protocols/aave.md), [Ethena](protocols/ethena.md) | Partner legibility and revenue share win deals |
| Launching a stablecoin or yield asset | [Sky](protocols/sky-maker.md), [Ethena](protocols/ethena.md), Aave GHO ([Aave](protocols/aave.md)) | Compete by shaping circulation |
| An institutional wrapper | [Lido](protocols/lido.md), Aave Horizon ([Aave](protocols/aave.md)), [Ondo](protocols/ondo-finance.md), [Maple](protocols/maple-finance.md) | Change the access path, not the product |
| Creating a new category | [EigenCloud](protocols/eigencloud.md), [Ethena](protocols/ethena.md) | Own the vocabulary; plan the pivot early |
| Escaping a commoditizing category | [ether.fi](protocols/etherfi.md), [EigenCloud](protocols/eigencloud.md), [Uniswap](protocols/uniswap.md) | Re-platform from strength, before churn |
| Building on a single chain ecosystem | [JustLend](protocols/justlend.md), [Spark](protocols/spark.md) | Sponsorship works but does not travel |
